# Система учёта заявок сотрудников

Веб-приложение для управления заявками сотрудников с фильтрацией, отчётностью и REST API.

## О проекте

Проект реализован в рамках тестового задания. Представляет собой полноценную систему учёта заявок с CRUD-операциями, справочником сотрудников, фильтрацией, пагинацией, отчётностью и валидацией бизнес-правил.

## Технологический стек

- Backend: Python 3.13, FastAPI
- ORM: SQLAlchemy 2.0
- База данных: PostgreSQL
- Шаблоны: Jinja2
- Frontend: HTML5, CSS3, Vanilla JavaScript
- ASGI-сервер: Uvicorn
- Генерация тестовых данных: Faker

## Запуск проекта

### 1. Клонирование репозитория

git clone https://github.com/your-username/employee_requests.git
cd employee_requests

### 2. Создание виртуального окружения

python -m venv venv
venv\Scripts\activate

### 3. Установка зависимостей

pip install -r requirements.txt

### 4. Настройка базы данных

Создайте файл .env в корне проекта:

DATABASE_URL=postgresql://user:password@localhost:5432/employee_requests

### 5. Создание таблиц и заполнение тестовыми данными

python seed_db.py

Скрипт создаст:
- 1000 сотрудников
- 1 000 000 заявок
- Индексы для оптимизации запросов

### 6. Запуск сервера

uvicorn app.main:app --reload

Приложение будет доступно по адресу: http://127.0.0.1:8000

Документация API (Swagger UI): http://127.0.0.1:8000/docs

## Функциональность

### Справочник сотрудников
- ФИО, подразделение, должность
- Добавление новых сотрудников
- Выпадающие списки из существующих значений

### Заявки
- Создание заявок с выбором автора и исполнителя
- Поля: номер, дата создания, автор, исполнитель, описание, срок, статус
- Автоматическая генерация номера (REQ-XXXXXXX)

### Работа с заявками
- Изменение статуса с проверкой допустимости переходов:
  - Новая → В работе, Выполнена
  - В работе → Выполнена
  - Выполнена — изменения запрещены
- Изменение исполнителя
- Фильтрация по статусу, исполнителю, подразделению, просроченным заявкам
- Бесконечная прокрутка (infinite scroll) с подгрузкой по 50 записей

### Отчётность
- Количество заявок по статусам
- Количество просроченных заявок
- Выполненные заявки по исполнителям

## База данных

### Нормализация
- 3-я нормальная форма (3НФ)
- Отдельные таблицы для сотрудников и заявок
- Внешние ключи для связей

### Индексы
- Составной индекс по (status, due_date) для ускорения фильтрации
- Индексы по внешним ключам author_id, assignee_id

### Производительность
- Замер производительности до и после оптимизации
- Пагинация на уровне SQL (LIMIT/OFFSET)

## API Endpoints

### Заявки

GET /api/requests/ - Список заявок с фильтрацией
POST /api/requests/ - Создание заявки
PATCH /api/requests/{id}/status - Изменение статуса
PATCH /api/requests/{id}/assignee - Изменение исполнителя
GET /api/requests/reports - Отчётность

### Сотрудники

GET /api/employees/ - Список сотрудников
POST /api/employees/ - Добавление сотрудника
GET /api/employees/{id} - Получение сотрудника
GET /api/employees/departments - Список подразделений
GET /api/employees/positions - Список должностей

## Тестирование

## Примечания

- Все данные генерируются библиотекой Faker (случайные ФИО, описания)
- При добавлении нового сотрудника список обновляется кнопкой "Обновить список сотрудников"
- Проект адаптирован под Windows (пути venv\Scripts\activate)
